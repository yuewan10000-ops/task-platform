# 推送代码到 GitHub - 解决方案

## 问题
网络无法直接连接到 GitHub（端口 443 连接失败）

## 解决方案（选择其一）

### 方案 1：使用 GitHub Desktop（最简单）

1. **下载 GitHub Desktop**
   - 访问：https://desktop.github.com/
   - 下载并安装

2. **登录 GitHub**
   - 打开 GitHub Desktop
   - 登录你的 GitHub 账号（yuewan10000-ops）

3. **添加本地仓库**
   - File → Add Local Repository
   - 选择：`C:\Users\admin\Desktop\task-platform`
   - 点击 "Add repository"

4. **推送代码**
   - 点击 "Publish repository"
   - 选择 "yuewan10000-ops/task-platform"
   - 点击 "Publish repository"

---

### 方案 2：配置 Git 代理（如果有代理）

如果你有可用的代理，可以配置 Git 使用代理：

```bash
# HTTP 代理
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 然后推送
git push -u origin main

# 推送完成后，取消代理
git config --global --unset http.proxy
git config --global --unset https.proxy
```

---

### 方案 3：使用 SSH 方式（如果 SSH 可用）

1. **生成 SSH Key（如果还没有）**
   ```bash
   ssh-keygen -t ed25519 -C "your.email@example.com"
   # 按 Enter 使用默认路径
   # 设置密码（可选）
   ```

2. **复制公钥**
   ```bash
   cat ~/.ssh/id_ed25519.pub
   # 复制输出的内容
   ```

3. **添加到 GitHub**
   - 访问：https://github.com/settings/keys
   - 点击 "New SSH key"
   - Title: 输入描述（如：My Computer）
   - Key: 粘贴刚才复制的公钥
   - 点击 "Add SSH key"

4. **修改远程仓库 URL 为 SSH**
   ```bash
   git remote set-url origin git@github.com:yuewan10000-ops/task-platform.git
   ```

5. **推送**
   ```bash
   git push -u origin main
   ```

---

### 方案 4：通过 GitHub 网页上传（临时方案）

1. **压缩项目文件**
   - 在 `C:\Users\admin\Desktop\task-platform` 目录
   - 选择所有文件（排除 `node_modules`、`.env`、`dist` 等）
   - 压缩为 ZIP 文件

2. **上传到 GitHub**
   - 访问：https://github.com/yuewan10000-ops/task-platform
   - 点击 "uploading an existing file"
   - 拖拽 ZIP 文件或选择文件
   - 点击 "Commit changes"

**注意：** 这种方式不会保留 Git 历史记录，但可以快速上传代码。

---

### 方案 5：使用 Gitee（国内 GitHub 替代品）

如果 GitHub 访问困难，可以使用 Gitee（码云）：

1. **注册 Gitee 账号**
   - 访问：https://gitee.com/signup

2. **创建仓库**
   - 访问：https://gitee.com/projects/new
   - 仓库名称：task-platform
   - 点击创建

3. **推送代码**
   ```bash
   git remote set-url origin https://gitee.com/你的用户名/task-platform.git
   git push -u origin main
   ```

---

## 推荐方案

**如果你有代理：** 使用方案 2（配置代理）
**如果没有代理：** 使用方案 1（GitHub Desktop）或方案 5（Gitee）
